Figma Variables Prep

Website design token starter

This form helps you define a small design token system for your website. You will use these values later when creating variables and styles in Figma.

Semantic naming is preferred because the role stays stable even if the actual font or color changes.

Choose fonts from Google Fonts when possible, since those are also commonly available in Figma. Write the font name with the exact spelling so your variable values match correctly later.

Choose accessible, readable text and background combinations so your website is easier to use.

In Figma, variables store values. Styles apply those values in actual text and UI elements.

How to use this

  1. Pick a small set of fonts and colors for your website.
  2. Use Google Fonts and record each font name exactly as written.
  3. Keep names role-based, like primary text or heading font.
  4. Copy the export list when you are ready to build in Figma.

Step 4

Build variables in Figma

1. Create collections

In Figma, open Variables and make a collection for typography tokens and another for color tokens.

2. Add token names exactly

Create each variable using the token name shown here, such as `font.family.heading` or `color.text.primary`.

3. Enter values carefully

Paste or type the matching value for each token. Font names should use the exact Google Fonts spelling, including spaces.

4. Turn variables into styles

After your variables are set, make text styles and color styles in Figma that use these variable values in real UI elements.

Likely Figma collections

Typography collection

Add your `font.family.*` values to a typography-related variable collection, then make text styles for headings and body text.

Color collection

Add your `color.*` values to a color variable collection, then apply them in color styles or directly in components.

Future collections

You can expand this system later with spacing, radius, shadow, or image treatment tokens.
