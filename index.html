Figma Variables Prep

Website design token starter

This form helps you define a small design token system for your website. You will use these values later when creating variables and styles in Figma.

NOTES:

Semantic names with slashes (eg font/family/brand) are used for understanding and to keep things organized in Figma.

Choose fonts from Google Fonts listed in the auto complete. Figma uses Google fonts and this form ensures the spelling is accurate. when possible, since those are also commonly available in Figma. Write the font name with the exact spelling so your variable values match correctly later.

Color contrast (between text and background color) must be accessible, Use step 3 to make sure your combinations are valid.

This workflow makes your project easier to update by storing the font names and color values in variables and then applying these variables to text and color styles. The form and associated plugin help you bring this advanced workflow to you projects quickly.

How to use this

Before you start, you should have done research and planning on the typography and colors for your site.

  1. Step 1: Set fonts and colors. Choose your main font families for your site name/brand, headings and body (base font). Google Fonts are used because Figma uses them too. Pick your colors. Note the roles provided (primary, secondary, text, background)
  2. Step 2: Review the summary to make sure you entered things correctly and that your text/background color combinations are accessible.
  3. Step 3: Copy the export tokens.
  4. Step 4: Run the Website Style Token Importer plugin and paste in the tokens you copied.

When you're done, you should have variables and styles defined in your Figma file.

Step 3

Export tokens

Plugin output overview

Typography variables and text styles

`font/family/*` tokens become typography variables, and the plugin also creates text styles that reference those variables.

Color variables and color styles

`color/*` tokens become color variables, and color styles are generated so those values can be applied in UI elements.

Extra tokens you add

If you add extra `font/` or `color/` slash-named token lines, those variables are created too.

Step 4

Run the plugin in Figma

1. Open the plugin

In your Figma file, run the Website Style Token Importer plugin from your class team plugins.

2. Paste Step 3 export and import

Copy the token list from Step 3, paste it into the plugin, then click Import.

3. What the plugin creates

The plugin creates variables first, then creates text and color styles that use those variables.

4. How to update values later

You can either fill out this form again and re-import, or edit variable values directly in Figma.

Side note: extra token lines

You can add more `name=value` lines before importing. If the token name starts with `font/` or `color/` and uses slash syntax, the plugin will create those variables too.
